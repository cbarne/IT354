Usability Mistakes

Tiny Clickable Areas

Content that is to small to click is not very accessible for users. The hand movement for users is not 100% accurate and some users have issues with seeing small things.


Page Titles

Page titles help determine what the user will find once the webpage has been search on engines like Google. If it is not a descriptive enough title and it is too generic. Users may never come across the webpage like was intended.


Content Scanning

First rule of web design is does it hurt to look at, is the layout confusing, and is this pleasing to look at? If the page displays items all over the user will not be able to understand how we have it set up. The user reads from top to bottom and the gaze generally zig zags. So focus points are good to have.

Principale of Website Usability

♦ 2 Second Rule: This rule pertains to the users attention span. If it will not load in two second, than the experience is garbage. So try to keep any type of application launch time under that.
♦ 3-Click Rule: The user should be able to access most if not all of the information the site needs to offer within three clicks. It shows that there is clear navigation and it shows there is not a mass of clutter.
♦ Inverted Pyramid: The summary is presented at the beginning of the article rather then the end. Since most users have the attention span of a gerbal, it allows for the meatier substance of information to be at the beginning.
♦ Don't Use Pop-Ups: It interupts the viewer from what they are doing and potentially nags them to want to leave.
♦ Don't Have Dead Links: The use of dead links hinders the user from what was meant to be, but never was. It makes them useless to even put in.
♦ Have Only One Animation Per Page: It can distract the user from the content that is on the page. Making them just as ADHD as kid off his meds trying to study.
♦ Make Contact Easy: Communication is key. It can lead to missed opportunities, no trust, or even worse the user won't return due to lack of support.
♦ Make Links Open on the Same Page: User has control and is like a customer. You let them decide what they want to do and not do it for them.
♦ Ad Banner Blindness: Nobody likes ad's. That includes banner ad's. They are noticed by people, but who really clicks on them nowadays? Nobody, that's who.
♦ Eye Tracking: It is how the user navigates a webpage just by looking at it. If it is difficult to follow the user will have a bad experience.

Node.Js The Javascript Engine

What is Node?

It is a open source, cross plateform Javascript runtime enviroment. It uses the javascript code outside of web browsers. It is built on chromes V8 engine. It is very fast in code execution. While also Node.js library being asynchronous allowing for non-blocking to happen. Since non-blocking occurs it also allows for no buffering. Which makes it useful for JSON Api applications and single page.


What does it mean by engine?

V8 Runtime Engine by Google Chrome is a runtime enviorment that contains the Javascript engine which executes script code. It basically converts the javascript code into a lower tier code that microprocessors can then understand. It is written in C++ while being used in chrome and NodeJs. While the V8 engine is running the platform API and DOM are both provided by the browser. It works independently from the browser that it is being hosted inside of. Making it a popular engine.


History of NodeJs

Javascript is 25 years old, and Node is only 11. When compared to the time the internet has been around (which is 31 years) that is 1/3 of its lifetime! In 2009 Node.js was first created. It took until 2011 to make it somewhat popular. That is when tutorials for it began to appear. By 2014 it was announced that there was a proposal to make Node.js go open source. By 2017 it had reached mainstream medias attention. It was huge for users and Node.js alike. During that time it reached more then 40k stars on github and 3 billion npm package downloads. By 2020 it has become a common used tool in almost every front for the workplace.
