Window Object

A window object is a literal representation of an open window in the browser. All global Javascript objects can automatically become members of the window object. Since global variables are the properties of the window object and the global functions are the methods to the window object.


Global Object

As stated before, global objects can automatically become part of a window object. They provide functions, and variables that are avaliable anywhere. Hence the name global. For the typical browser it is known as window since it can be inside of the window object, but for Node.js it is global.



MicroServices Vs Monolith

The idea is that Microservices are the cool new thing and are outplaying the old Monolith technology. Since there are lots of difficultys that follow with Monolith. Some of those being, "huge code base, adopting a new technology, scaling, deployment, implementing new changes and others". As it is the traditional way to develop apps there was bound to be a challenger eventually.

Monolith is the traditional approach at building the single unit. The issue with it being a single unit is if an update must occur, it will affect the whole code stack. Not just one or two things. It does have some positives though which include its debugging and testing capabilities. It becomes simple to deploy since it is just one single unit rather than multiple. However it does face the new technology barriers that have arose. Including the ability to scale. Which also hinders making changes since it is so large.

However Microservices are not perfect either. While it looks pretty on the outside, does it do the job? It starts by breaking the application down into smaller units that are all independant rather then one big one. So deployables become quick and simple and are able to communicate with each other through the use of API's. Which helps with the ease of understanding. Creating a better ability for scaling to occur with still being independant. However it is very complex and the testing of such a system is very hard to do.

Which one is better you ask? Well both actually. Just depends on what kind of business you plan to do and what you wish to achieve.
